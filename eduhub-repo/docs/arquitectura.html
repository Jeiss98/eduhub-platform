01

Problema y Solución

Tienes 3 proyectos de distintas materias con requerimientos técnicos diferentes. La solución es construir una sola plataforma que los integre de forma coherente, justificando técnicamente el uso de dos bases de datos distintas.

¿Por qué funciona esta integración?

🏫 Contexto real

Todos los portales universitarios reales tienen: gestión académica + noticias/anuncios. Es natural y coherente.

🗃️ Justificación técnica

Los datos académicos son estructurados (relacional). Las noticias son contenido flexible y variable (no relacional).

05

Módulos del Sistema

🔐 Autenticación

Login, registro, recuperación de contraseña. Roles: Estudiante, Docente, Administrador. JWT con expiración.

📁 Proyectos

CRUD de proyectos académicos. Asignación de estudiantes. Estado: activo, pausado, finalizado.

✅ Tareas

Subtareas dentro de proyectos. Asignación individual. Prioridad y fecha límite. Cambio de estado.

📊 Dashboard

Gráficos de progreso por proyecto. Tareas completadas vs pendientes. Estadísticas del usuario.

📰 Noticias

Portal de anuncios y noticias institucionales. Categorías, etiquetas, búsqueda. Administrado por docentes/admin.

📧 Notificaciones

Emails automáticos al asignar tareas, fechas próximas a vencer, y cuando se publica una noticia relevante.

Plan de Trabajo Sugerido

Semana 1
Configuración base
Setup Node.js + Express. Conexión dual MySQL + MongoDB. Modelos y migraciones. Variables de entorno.
Semana 2
Autenticación + API Core
Registro/login con JWT. CRUD de proyectos y tareas. Rutas protegidas por rol. Pruebas con Postman.
Semana 3
Módulo de Noticias + Emails
API de noticias con MongoDB. Búsqueda y filtros. Integración Nodemailer. Notificaciones automáticas.
Semana 4
Frontend React
Login, dashboard con gráficos (Recharts), vistas por rol, portal de noticias, componentes responsivos.
Semana 5
Pulido y Despliegue
Pruebas de integración, corrección de bugs, despliegue backend en Railway, frontend en Vercel/Netlify.
